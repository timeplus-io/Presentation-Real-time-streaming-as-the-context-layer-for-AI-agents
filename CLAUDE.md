## Project

A 15-slide interactive presentation ("Real-time Streaming as the Context Layer for AI Agents") built as a single-page React app. There is no backend, no test suite, and no router — the entire deck is rendered from `src/App.tsx`.

## Commands

- `npm run dev` — Vite dev server on `http://localhost:3000`, bound to `0.0.0.0`.
- `npm run build` — standard multi-file build to `dist/`.
- `npm run build:all` — **single-file** build (via `vite-plugin-singlefile`) to `docs/`. This is the distributable artifact (an HTML file with everything inlined); rebuild it whenever slide content changes that should be shipped.
- `npm run lint` — runs `tsc --noEmit`. There is no ESLint config; type-checking is the only static check.
- `npm run preview` — preview a production build.
- `npm run clean` — `rm -rf dist`.

There are no unit tests. Verify changes by running `npm run dev` and clicking through the affected slides.

## Architecture

### The deck is one file
`src/App.tsx` defines all 15 slides inline as children of `<Deck>`. Each `<div>` child is one slide. To add/remove/reorder slides, edit the children array in `App.tsx` — there is no slide registry or routing layer. The `{/* Slide N: ... */}` comments are the only index.

Two large inline subcomponents (`StreamingPipelineDiagram`, `XebiaArchitectureDiagram`) live at the top of `App.tsx` rather than under `src/components/`. Keep diagrams that are used by exactly one slide local to `App.tsx`; promote to `src/components/` only when reused.

### Fixed-canvas scaling
`src/components/Deck.tsx` renders every slide at a fixed `1024×576` logical viewport, then applies a CSS `transform: scale()` to fit the window (capped at `1`). All slide layouts assume this canvas size — using `vh`/`vw` units or assuming the real viewport size will break the scaling. Slides receive keyboard navigation: ArrowLeft / ArrowRight / Space.

### Animation library
Imports come from `motion/react` (the new package name for Framer Motion v12), not `framer-motion`. Match the existing import style.

### Auto-generated icons
`src/components/TechIcons.tsx` is generated by `fetch_icons.cjs` (or `.js`), which downloads SVGs from the `simple-icons` repo. Don't hand-edit `TechIcons.tsx` — regenerate via `node fetch_icons.cjs` if the icon set changes.

### Vite config gotchas (`vite.config.ts`)
- `@/*` is aliased to the repo root (not `src/`).
- `GEMINI_API_KEY` from `.env` is inlined into the bundle via `define`. Do not commit `.env`.
- The `mode === 'all-in-one'` branch swaps in `viteSingleFile` and changes `outDir` to `docs/`. The two builds (`dist/` vs `docs/`) are intentional and serve different purposes — the all-in-one build is what gets distributed standalone.
- The comment on the `server.hmr` line says "do not modify — file watching is disabled to prevent flickering during agent edits" when `DISABLE_HMR=true`. Respect this; the deck contains many `motion` animations that re-trigger on hot reload.
